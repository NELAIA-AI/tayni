# TAYNI Public Roadmap

> **TAYNI**: AI-first programming language designed for optimal token efficiency and minimal binary sizes.

## Current Status: Alpha

TAYNI is in active development. Features and timelines may change.

---

## Completed

- **Core Compiler**: Generates native Windows PE executables
- **AI-Native Syntax**: Optimized for token efficiency
- **Direct Syscalls**: No runtime dependencies
- **Standard Library (TIER 0)**: 10 essential modules
  - `file` - File I/O
  - `string` - String manipulation
  - `json` - JSON parsing
  - `http` - HTTP parsing
  - `url` - URL handling
  - `router` - URL routing
  - `log` - Logging
  - `base64` - Base64 encoding
  - `random` - Random number generation
  - `args` - CLI arguments

---

## In Progress

- **Self-Hosting**: TAYNI compiler written in TAYNI
- **Standard Library (TIER 1)**: Additional common modules

---

## Planned

- **Multi-Target**: WASM, ARM64, RISC-V support
- **Standard Library (TIER 2)**: Specialized modules (crypto, database, etc.)
- **Package System**: Module distribution

---

## Design Principles

1. **Token Efficient**: Minimal syntax for AI consumption
2. **Zero Dependencies**: Single executable output
3. **Declarative**: Data flow over control flow
4. **Native Performance**: Direct machine code generation
